# 棒球挥棒动作 Pose 分析模型训练 PRD

## 1. 项目概述

### 1.1 背景
本项目旨在通过计算机视觉和姿态估计技术，对棒球挥棒动作进行精准分析。通过训练专用的 2D 和 3D Pose 模型，实现对棒球挥棒动作的连续追踪与三维重建，为后续的动作评估、训练反馈和生物力学分析提供数据基础。

### 1.2 目标
- 构建能够精准识别棒球挥棒动作的 **2D Pose 模型**
- 将 2D 动作数据升维为 **3D Pose 模型**，实现三维空间中的动作重建
- 通过手工微调确保输出轨迹的连贯性和准确性
- 利用微调后的高质量数据反向优化模型，提升模型在棒球挥棒场景下的表现

### 1.3 数据资源
- **训练视频集**：包含标准、完整的棒球挥棒打击动作视频
- **验证视频集**：存放于独立目录，用于模型训练后的效果验证

## 2. 功能需求

### 2.1 2D Pose 动作识别与微调

#### 2.1.1 动作识别 (a)
- 使用现有的 2D Pose 动作追踪模型（如 MediaPipe Pose、OpenPose、YOLO-Pose 或 RTMPose）对训练视频进行逐帧分析。
- 输出每一帧中人体关键点的 2D 坐标（至少包含与挥棒动作相关的主要关节：手腕、手肘、肩膀、髋部、膝盖、脚踝、脊椎、头部等）。
- 将识别结果以结构化的序列数据格式保存（如 JSON 或 NumPy 数组），便于后续处理。

#### 2.1.2 轨迹点分析 (b)
- 由于训练视频中的挥棒动作是标准且完整的，需要对模型识别出的关键点轨迹进行分析。
- 检测每一点在时序上的变化规律，识别出与正常挥棒轨迹偏离较大的帧。
- 重点关注挥棒过程中的高动态区域：双手握棒位置、肩部旋转、髋部扭转等。

#### 2.1.3 跳变点修正 (c)
- 设计算法或人工检查机制，找出轨迹中的**突然跳变点**（如前后帧坐标差异过大、轨迹不连贯、关键点丢失或错位等）。
- 对跳变点进行插值平滑或手动校正，确保最终捕捉到的 2D 轨迹是一条连贯、完整的棒球挥棒动作曲线。
- 记录所有修正操作及修正原因，形成数据清洗日志。

#### 2.1.4 反向微调 2D Pose 模型 (d)
- 使用修正后的高质量 2D 轨迹点作为**Ground Truth**。
- 对原始 2D Pose 模型进行**有监督微调（Fine-tuning）**，使模型在棒球挥棒数据集上的输出更加稳定和准确。
- 微调策略包括但不限于：
  - 冻结部分骨干网络层，仅微调高层回归头
  - 针对棒球挥棒动作设计数据增强策略（时序采样、角度旋转、亮度变化等）
  - 使用 L1 Loss 或 Smooth L1 Loss 降低跳变误差

#### 2.1.5 验证集测试 (e)
- 使用验证目录下的视频对微调后的 2D Pose 模型进行推理测试。
- 评估指标：
  - **时序连贯性（Temporal Consistency）**：相邻帧关键点位移的平滑程度
  - **关键点准确性（Keypoint Accuracy）**：与预期标准动作的对齐程度
  - **漏检率 / 误检率**：关键点的可见性与稳定性
- 输出验证报告，若结果不达标则返回步骤 2.1.3 进行进一步数据清洗和再训练。

---

### 2.2 3D Pose 模型转换与优化

#### 2.2.1 2D 转 3D Pose (a)
- 基于 2.1.4 微调后的 2D Pose 结果，使用 3D Pose  lifting/estimation 模型（如 VideoPose3D、MotionBERT、DWPose-3D 或自定义网络）将 2D 坐标转换为 3D 坐标。
- 利用训练视频的多视角信息（如有）或单目深度估计技术，为 3D 重建提供空间约束。
- 输出每一帧的 3D 关键点坐标序列，表示人体在三维空间中的姿态。

#### 2.2.2 逐帧微调 (b)
- 逐帧读取生成的 3D 动作点序列，检查动作在三维空间中的连贯性。
- 检测并标记以下异常：
  - 关节长度在时序上的剧烈变化（违反骨骼长度约束）
  - 不合理的关节角度（如肘部反折、膝盖过度内扣）
  - 空间位置跳变（人物在三维空间中瞬移）
  - 与地面穿透或浮空现象

#### 2.2.3 手动校正 (c)
- 对于检测到的异常帧，开发可视化工具（如基于 Three.js / Matplotlib / Open3D 的 3D 骨架播放器），允许逐帧查看并手动拖动关键点至正确位置。
- 校正原则：
  - 保持骨骼长度在时序上的一致性
  - 确保挥棒动作符合人体工学和棒球发力链（Kinetic Chain）规律
  - 确保头部、躯干、髋部、手臂、球棒的相对位置合理
- 手动校正后的 3D 轨迹需经过二次审核，确保整条动作序列的连贯性和完整性。

#### 2.2.4 反向训练 3D Pose 模型 (d)
- 将手动校正后的高质量 3D 轨迹作为新的 Ground Truth。
- 对 3D Pose 模型进行端到端微调或重新训练，目标包括：
  - 提升模型在棒球动作场景下的 3D 重建精度
  - 增强输出结果的时序稳定性和骨骼约束满足度
  - 减少需要手动校正的异常帧比例
- 引入骨骼长度约束损失（Bone Length Loss）和时序平滑损失（Temporal Smoothness Loss）作为正则化项。

### 2.3 结果验证

#### 2.3.1 验证流程
- 使用验证目录下的独立视频，分别输入到微调后的 **2D Pose 模型**和 **3D Pose 模型**中。
- 对验证集的输出结果进行定量和定性评估。

#### 2.3.2 评估指标

**2D 输出评估：**
- 时序平滑度（Temporal Smoothness Score）
- 关键点稳定性（Keypoint Stability）
- 挥棒动作完整性（Swing Completeness）

**3D 输出评估：**
- 3D 轨迹连贯性（3D Trajectory Coherence）
- 骨骼长度一致性（Bone Length Consistency）
- 物理合理性（Physical Plausibility）
- 与标准挥棒动作的姿态相似度（Pose Similarity）

#### 2.3.3 验收标准
- 2D Pose 输出在验证视频上的跳变帧比例 **< 5%**
- 3D Pose 输出的骨骼长度时序变化标准差 **< 阈值（待标定）**
- 验证集上的整体动作序列无需大规模手动修正即可达到可直接使用的质量标准
- 输出节奏与实际挥棒节奏一致（无明显滞后或超前）

## 3. 非功能需求

### 3.1 性能要求
- 单帧 2D Pose 推理耗时 **< 50ms**（在目标硬件上）
- 单段视频（约 3-5 秒，90-150 帧）的全流程处理（2D + 3D）**< 2 分钟**

### 3.2 数据管理
- 训练数据、中间结果、模型权重、验证报告需按规范目录存放
- 所有人工修正的数据需保留版本记录，支持回溯

### 3.3 可扩展性
- 框架设计需预留接口，未来可扩展至其他棒球动作（投球、接球、跑垒）或其他运动项目

### 3.4 可视化
- 提供 2D 和 3D 结果的可视化回放工具
- 支持叠加原始视频与 Pose 骨架
- 支持慢速播放、逐帧步进、关键帧标记

## 5. 风险与依赖

### 5.1 技术风险
- **2D 跳变严重**：如果原始 2D 模型在挥棒场景下表现极差，可能导致手工修正工作量过大
- **3D 重建歧义**：单目视频升维 3D 存在深度歧义，可能需要多视角数据或引入先验约束
- **数据量不足**：训练视频数量若过少，微调可能导致过拟合

### 5.2 依赖项
- 训练视频和验证视频的获取与标注质量
- 目标部署硬件环境（GPU 型号、显存大小）
- 可视化工具的开发时间

### 5.3 缓解方案
- 在数据清洗阶段投入足够时间，建立半自动化的异常检测 pipeline
- 考虑引入 SMPL 或棒球专用人体先验模型
- 若视频数量不足，结合时序插值、镜像翻转、速度变化等数据增强手段扩充数据集

## 6. 附录

### 6.1 关键术语
- **Pose Estimation**：姿态估计，从图像/视频中提取人体关键点位置的技术
- **Fine-tuning**：微调，在预训练模型的基础上，使用领域特定数据继续训练
- **Lifting**：升维，将 2D 坐标映射到 3D 空间的过程
- **Kinetic Chain**：发力链，棒球挥棒时力量从下肢经躯干传递至上肢和球棒的连贯过程

### 6.2 参考模型
- 2D Pose：RTMPose、YOLO-Pose、MediaPipe Pose
- 3D Pose：VideoPose3D、MotionBERT、DWPose-3D
