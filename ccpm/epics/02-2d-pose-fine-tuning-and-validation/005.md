## Description

Build a report generator that consumes the validation metrics JSON and produces a human-readable markdown report at `reports/2d_validation_report.md`, including per-video breakdowns and aggregate statistics.

## Acceptance Criteria

- [ ] Script `scripts/generate_validation_report.py` exists and accepts `--metrics-json` and `--output` arguments.
- [ ] Report `reports/2d_validation_report.md` is generated after every validation run.
- [ ] Report contains a summary table with aggregate TC, KS, and Missing Rate, and a pass/fail indicator for each target threshold.
- [ ] Report lists per-video metrics in a markdown table (video filename, TC, KS, Missing Rate).
- [ ] Report includes the training config snapshot path and the fine-tuned checkpoint path for traceability.
- [ ] Report header contains generation timestamp in UTC ISO 8601 format.
- [ ] Running the script with the same metrics JSON produces deterministic output (no random values).
