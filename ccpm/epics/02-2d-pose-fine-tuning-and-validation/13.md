## Description

Implement the training script that loads the prepared dataset and config, fine-tunes RTMPose-m, and enforces locked hyperparameters with early stopping. The script must combine Smooth L1 Loss with a temporal consistency loss and support autonomous ReduceLROnPlateau when validation loss plateaus.

## Acceptance Criteria

- [ ] Script `scripts/train_2d_pose.py` exists and can be invoked from the project root.
- [ ] Loss function computes `SmoothL1Loss` on keypoint coordinates plus an L1 temporal consistency term `|pred[t] - pred[t-1]|` averaged over the batch.
- [ ] Training stops automatically if validation loss does not improve for 10 consecutive epochs (early stopping patience=10).
- [ ] ReduceLROnPlateau is triggered automatically when validation loss plateaus for 5 epochs, reducing LR by factor 0.1, bounded within `[1e-5, 1e-3]`.
- [ ] Best model checkpoint (by validation loss) is saved to `models/2d_pose/rtmpose_m_finetuned.pth`.
- [ ] Training logs (loss curves, LR history, epoch times) are written to `logs/training_logs/training_log.json` or equivalent structured format.
- [ ] Script completes without out-of-memory errors on a mocked 24 GB VRAM budget (or verified on actual RTX 4090 if available).
- [ ] No manual CLI flags allow changing locked hyperparameters (freeze stages, initial LR, batch size, weight decay, max epochs).
