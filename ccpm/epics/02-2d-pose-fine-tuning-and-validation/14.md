## Description

Create a validation/inference script that runs the fine-tuned RTMPose-m model on the held-out validation set and computes the three required quantitative metrics: Temporal Coherence (TC), Keypoint Stability (KS), and Missing Rate.

## Acceptance Criteria

- [ ] Script `scripts/validate_2d_pose.py` exists and accepts `--model` and `--val-dir` arguments.
- [ ] Script loads `models/2d_pose/rtmpose_m_finetuned.pth` and runs frame-by-frame inference on all videos in `data/raw_videos/val/` (or prepared val frames).
- [ ] Temporal Coherence (TC) is computed per video as `1 / (1 + mean(||P_t - 2P_{t-1} + P_{t-2}||))` and aggregated across all videos.
- [ ] Keypoint Stability (KS) is computed per video as `1 - (jump_frames / total_frames)` where a jump frame has any keypoint displacement > `mu + 3*sigma`, and then aggregated.
- [ ] Missing Rate is computed as the proportion of keypoints with model confidence < 0.3, averaged across frames and videos.
- [ ] Script outputs a JSON file `logs/validation_metrics.json` containing per-video and aggregate values for TC, KS, and Missing Rate.
- [ ] Aggregate validation results meet targets: TC > 0.82, KS > 0.92, Missing Rate < 0.05.
