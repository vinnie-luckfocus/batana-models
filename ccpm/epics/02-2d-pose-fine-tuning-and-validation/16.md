## Description

Implement an autonomous self-correction loop that detects validation failure (any metric below target) and automatically triggers Phase 01 re-cleaning with adjusted parameters, then re-trains and re-validates without human intervention.

## Acceptance Criteria

- [ ] Orchestrator script `scripts/run_epic_02_pipeline.py` exists and wraps dataset prep, training, validation, and reporting.
- [ ] After validation, if TC <= 0.82 or KS <= 0.92 or Missing Rate >= 0.05, the orchestrator detects failure and logs `VALIDATION_FAILED` to `logs/pipeline.log`.
- [ ] On failure, the orchestrator invokes the Phase 01 cleaning script (or equivalent entry point) with a stricter correction threshold flag (e.g., `--strict-mode`).
- [ ] The orchestrator re-runs dataset preparation (11), training (13), and validation (14) using the newly cleaned outputs.
- [ ] At least one full self-correction loop is attempted before the pipeline exits.
- [ ] If the loop succeeds on retry, the final report reflects the successful metrics and checkpoint.
- [ ] If the loop still fails after one retry, the pipeline exits with code 1 and writes `SELF_CORRECTION_EXHAUSTED` to `logs/pipeline.log`.
- [ ] All loop iterations are tracked in `logs/pipeline.log` with timestamps.
