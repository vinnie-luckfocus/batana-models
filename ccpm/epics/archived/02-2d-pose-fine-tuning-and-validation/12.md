## Description

Build a reproducible MMPose configuration file for fine-tuning RTMPose-m on the baseball swing dataset. All hyperparameters are locked as specified in the PRD. The config must include the custom 19-keypoint head, frozen backbone stages 0-2, and the required augmentations.

## Acceptance Criteria

- [ ] Config file `configs/rtmpose_m_finetune_baseball.py` exists and is parseable by MMPose.
- [ ] Model definition uses `rtmpose-m` backbone with the first 3 stages frozen (`frozen_stages=3` or equivalent).
- [ ] Keypoint head output dimension is 19 (coco-17-plus-bat).
- [ ] Optimizer is `AdamW` with `lr=5e-4`, `weight_decay=1e-4`.
- [ ] LR scheduler is `CosineAnnealingLR` with `T_max=50` (max epochs).
- [ ] Batch size is 16 across the training data loader.
- [ ] Data augmentation pipeline includes: random rotation ±30°, perspective ±15%, brightness ±30%, contrast ±20%, saturation ±20%, Gaussian noise sigma=5, horizontal flip, temporal downsampling to 15 fps, and Mosaic/MixUp if the MMPose version supports them.
- [ ] A separate config snapshot is written to `logs/training_logs/rtmpose_m_finetune_baseball_snapshot.py` at the start of every training run.
